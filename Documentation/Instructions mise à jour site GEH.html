Instructions pour la mise à jour après une partie


Prérequis

avoir GitKraken et Rstudio.


Mise à jour du fichier avec le pointage

Fichier à mettre à jour avec le pointage de la partie

https://github.com/gehstatcan/gehstatcan.github.io/blob/main/Saison23/Donn%C3%A9esCSV/tblR%C3%A9pondantsParJoueurs.csv


Pour faciliter la tâche, j’ai créer le fichier Excel (optionnel mais ça va bien en titi)

https://github.com/gehstatcan/gehstatcan.github.io/blob/main/Saison23/Donn%C3%A9esCSV/Saisie%20des%20points.xlsx


Il faut toutefois s’assurer que les onglets tblÉquipes et tblJoueurs soient à jour. Ce n’est pas le cas s’il y a eu des nouveaux joueurs…


Les étapes

  1. Pull https://github.com/gehstatcan/gehstatcan.github.io

  2. Mon répretoire local  :"C:\GIT\gehstatcan.github.io\

  3. ouvrir "C:\GIT\gehstatcan.github.io\Saison23\DonnéesCSV\Saisie des points.xlsx"

  4. S'assurer que les onglets tblÉquipes et tblJoueurs soient à jour.

  5. Ouvrir Saisie des points.xlsx

  6. Onglet Saisie

  7. Entrer le # de partie dans B2 et les noms d'équipes dans B3 et B4.

  8. Entrer les points par joueurs dans D8 à D33. S'assurer de vider les points avant.

  9. Copier K4 à K13 dans "C:\GIT\gehstatcan.github.io\Saison23\DonnéesCSV\tblRépondantsParJoueurs.csv"

  10. Si le fichier n'existe pas (parties 1 et 2), le copier d'une saison précédente et conserver les en-têtes.

Mise à jour du site web

Mes répertoires

Pour simplifier, voici les répertoires que j’utilisent sur mon ordi. Ce sont toutefois des paramètres modifiables dans le fichier _variables.yml

Mon exemple est pour la saison 23!


répertoire_données: "C:/GIT/gehstatcan.github.io/Saison23/DonnéesCSV"
répertoire_code: "F:/Users/MM/Documents/Git/CreationPageWeb/Quarto/"


Les étapes

  1. Ouvrir le projet https://github.com/gehstatcan/gehstatcan.github.io/ avec GitKraken

  2. Récupérer les fichiers les plus récent avec un PULL.

    1. Le but est d’avoir les CSV les plus récent.

    2. Moi, ça va dans C:/GIT/gehstatcan.github.io/...

  3. Ouvrir le projet https://github.com/gehstatcan/gehstatcan.github.io/ avec GitKraken

  4. Récupérer les fichiers les plus récent avec un PULL.

    1. C’est ce projet qui génére le site WEB

  5. ouvrir "...\CreationPageWeb\Quarto\GEHStatcanQMD.Rproj" avec Rstudio ou autre éditeur

  6. Ajuster le fichier _variables.yml pour votre répertoire et la saison en cours.

  7. Menu Build | Render web site

  8. En présaison, le fichier pointsparjoueur ne doit pas être là. C'est ce qui fait que le site ne va pas générer de classements.

  9. Copier de

F:\Users\MM\Documents\Git\CreationPageWeb\Quarto\sitewebracine\saison23

vers

C:\GIT\gehstatcan.github.io\Saison23\SaisonReg

  1. De GitKraken, faire un Commit et un PUSH du projet https://github.com/gehstatcan/gehstatcan.github.io

Sauvegarde des questionnaires sur le site

  1. Ouvrir le questionnaire en Excel, le sauvegarder en format HTM/HTML sous C:\GIT\gehstatcan.github.io\Saison23\SaisonReg\Questionnaires\

  2. De gitkraken (ou autre), pousser vers github  (commit, push...)
